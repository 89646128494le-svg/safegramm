
# Безопасность (каркас)

- Примитивы: ECDH P‑256 (WebCrypto) для выработки ключа, AES‑GCM для шифрования сообщений (DM).
- Ключи пользователей:
  - приватный ключ хранится локально в браузере (MVP). План: шифровать PIN‑паролем и хранить в secure storage.
  - публичный ключ публикуется на сервере.
- Сервер хранит только ciphertext и метаданные (кто кому и когда).
- Группы/каналы: пока без E2EE. План: MLS / Double Ratchet.
- Обязателен аудит перед реальным релизом.
